# Curbly Architecture\n\nFrontend (React, Vercel) → Backend (Node.js, Render) → Database (MongoDB)\n\n## Data Models\n\nUsers: id, email, passwordHash, role, subscription\nJobs: customerId, haulerId, status, address, distance, price, photoLocations\nDistanceHistory: estimated/actual distance, variance, confidence\nPayments: jobId, amount, status, stripeId\nAuditLogs: userId, action, resource, timestamp (90-day TTL)\n\n## GPS Distance Validation\n\nHaversine: Calculate actual distance from GPS coordinates\nVariance: Track estimated vs. actual (overpayment/underpayment alerts)\nConfidence: (positive_samples / total) * 0.9 + 0.1\nBidirectional Alerts: Flag both over- and under-estimation\n\n## Security\n\nPassword: Bcrypt hashing\nPayments: Stripe only (never store card data)\nGDPR: 90-day log retention, self-service deletion\nAccess Control: Strict state machine validation, server-side checks\n\n## Deployment\n\nFrontend (Vercel): $0 | Backend (Render): $7/mo | Database (MongoDB): $0
